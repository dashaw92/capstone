## Pantry inventory backend
This is the implementation for the backend server. Three "modules" comprise this project:

gRPC server: `bin/backend.dart`  
PostgreSQL: `bin/database.dart`, `bin/crud.dart`, `bin/database.g.dart` (generated code)  
Extractor FFI: `bin/extractors.dart`


## Dependencies  

### PostgreSQL
The server requires a PostgreSQL server to connect to upon startup. For my purposes, this is hardcoded to be at `localhost:5432` (provided via the scripts in this directory).

Prepare the PostgreSQL Docker container:
```sh
./init_db.sh
```

Start the container:
```sh
./start_db_container.sh
```

When finished, the container may be killed with the `./kill_db.sh` script.

### Protocol project (../pantry_protocol)
This project defines the common API between the client and server. It is automatically generated by protoc. See the protocol directory for more information.

### Extractors  
Extractors must be placed in the `./extractors` directory. Extractors may be symlinks to ease development iteration cycles. See `../extractors` for more information.

## Running  
Once the PostgreSQL container is running, starting the server can be achieved with the `dart run` command in this directory. The server is hardcoded to listen on port `8080`. Extractors will be registered upon startup and logged to output with their corresponding URLs. Live API calls will be logged as clients interact with the server. To terminate the server, just press `^C` in the running process.
